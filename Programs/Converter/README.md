## Задача: Разработка преобразователя файлов в байтовый
поток и обратно

### Описание
Требуется разработать консольное приложение, способное
преобразовывать файлы в байтовый поток и восстанавливать
файлы из этого потока обратно. Программа должна
поддерживать как обработку отдельных файлов, так и работу
с целыми каталогами. При обработке каталога, должна быть
возможность задания фильтра типов файлов для обработки.
Выходным результатом всегда должен быть один файл -
"коробка".

### Требования
1. Разработать функциональность преобразования файлов в
   байтовый поток и обратно.
2. Обеспечить возможность работы как с отдельными файлами,
   так и с каталогами.
3. Предусмотреть фильтрацию типов файлов при обработке
   каталога.
4. Выходным результатом всегда должен быть один файл -
   "коробка".
5. Реализовать шаблоны преобразования для каждого файла.

### Примерные шаблоны преобразования
Каждый порожденный файл должен содержать три сущности:
1. **Заголовок** (48 байт):
   - 4 байта: идентификатор файла.
   - 4 байта: тип коробки.
   - 8 байт: пустые (дополнительные).
   - 16 байт: список ключей файлов и смещение на блок
     конкретного ключа.
   - 8 байт: длина всего файла (без заголовка), разделенная
     на длину блоков для определения количества блоков.
   - 8 байт: пустые (дополнительные).

2. **Блок** (40 байт):
   - 8 байт: пустые (дополнительные).
   - 4 байта: смещение на ключ.
   - 4 байта: смещение на длину ключа.
   - 8 байт: пустые (дополнительные).
   - 4 байта: смещение на значение.
   - 4 байта: смещение на длину значения.
   - 8 байт: пустые (дополнительные).

3. **Ресурсы файла** (x байт):
   - x байт: последовательно записанные байты.

### Примеры
1. Преобразование одного файла: `Converter.exe --convert
   file.txt`
2. Преобразование каталога с фильтром: `Converter.exe
   --convert directory/ --filter *.xml`
3. Восстановление файла из байтового потока: `Converter.exe
   --revert box.byte`

### Замечания
- Предоставьте подробную инструкцию по запуску и установке
  необходимых зависимостей для использования приложения.
- Рассмотрите возможность передачи параметров преобразования
  через аргументы командной строки.
- Реализация шаблонов преобразования может потребовать
  детальной обработки и корректной структуры данных внутри
  файлов.
